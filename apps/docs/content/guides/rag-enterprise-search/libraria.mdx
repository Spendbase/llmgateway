Libraria is an AI chatbot builder that lets you create custom knowledge-based chatbots trained on your own data. It supports document ingestion, website scraping, and integration with various AI models for generating accurate, contextual responses.

Libraria supports custom API endpoints, allowing you to connect LLM API as the AI model backend for your chatbots.

## Prerequisites

- An LLM API account with an API key
- Libraria installed or accessible

## Setup

<Steps>
<Step>
### Get Your LLM API Key

1. Log in to your [LLM API dashboard](https://app.llmapi.ai/api-keys)
2. Click **Create Key to Start**
3. Copy your new API key immediately — it will only be shown once
4. Store the key securely (e.g., in a password manager or `.env` file)

<Callout type="info">
	LLM API is an OpenAI-compatible gateway that gives you access to dozens of AI
	models through a single API key and endpoint.
</Callout>

</Step>

<Step>
### Configure LLM API in Libraria

1. Log in to your Libraria dashboard.
2. Navigate to Settings → API Configuration or Model Settings.
3. Select "Custom" or "OpenAI Compatible" as the model provider.
4. Enter the following details:
   - **API Key**: paste the key you copied from app.llmapi.ai/api-keys
   - **Base URL**: https://api.llmapi.ai/v1
   - **Model**: the model ID you wish to use (e.g., openai/gpt-4o)
5. Click "Save" to store the configuration.
6. Your Libraria chatbots will now use LLM API for generating responses.

</Step>

<Step>
### Test the Integration

Verify that Libraria can successfully communicate with LLM API by sending a test request. All requests will now be routed through LLM API.

</Step>
</Steps>

<Callout type="info">
	Using a powerful model through LLM API improves the accuracy and quality of
	your Libraria chatbot's answers.
</Callout>

## Benefits of Using LLM API with Libraria

- **Multi-Provider Access**: Use models from OpenAI, Anthropic, Google, and more through a single API
- **Cost Control**: Track and limit your AI spending with detailed usage analytics
- **Unified Billing**: One account for all providers instead of managing multiple API keys
- **Caching**: Reduce costs with response caching for repeated requests

<Callout type="info">
	View all available models on the [models page](https://llmapi.ai/models).
</Callout>
